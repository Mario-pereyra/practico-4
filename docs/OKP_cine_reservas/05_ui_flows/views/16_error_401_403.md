# Errores 401 y 403

## Objetivo

Explicar claramente cuando el usuario necesita iniciar sesión o no tiene permisos suficientes.

## Casos

- 401: usuario no autenticado intenta reservar o entrar a vista privada.
- 403: cliente intenta entrar a admin.

## Layout

- `AppShell` si es público, `AdminShell` si corresponde.
- Card centrada con título, explicación y CTA.

## Mensajes

401:

> Necesitas iniciar sesión para continuar.

CTA: “Iniciar sesión”.

403:

> No tienes permisos para acceder a esta sección.

CTA: “Volver a cartelera”.

## Reglas UX

- No mostrar detalles técnicos.
- No revelar rutas internas.
- El usuario debe tener una acción clara.

## Prompt Stitch recomendado

Crear una pantalla de acceso restringido para una app web de cine. Mostrar una tarjeta centrada con mensaje claro para iniciar sesión o falta de permisos, un botón principal para volver o iniciar sesión, estilo sobrio y sin detalles técnicos.

## Checklist de aceptación

- Diferencia 401 y 403.
- Ofrece CTA claro.
- No expone información técnica.
